SVG — это как отдельная HTML-страница.
Как добавить SVG в HTML страницу:
  • object
    позволяет применять скрипты. Это самый надежный метод для обеспечения интерактивности.
    <object type="image/svg+xml" data="picture.svg">
    <img src="picture.png">
    </object>
    Pros! из соображений безопасности нельзя использовать картинку с другого домена
    Cons! CSS стили придется прописывать в SVG файл.
    Запуск SVG анимации.
  • img
    используется когда нет необходимости добавлять интерактивность.
    <img src="fallback.png" srcset="equilateral.svg">
    Pros! "srcset": older browsers will load the PNG instead SVG.
    Cons! SVG can't be manipulated with JavaScript.
  • css background image.
    используется когда нет необходимости добавлять интерактивность.
    {background: url("fallback.png") no-repeat center;
    background-image: url("image.svg");
    background-size: contain;}
    Pros! use fallback - older browsers will stick with the PNG that they understand, while newer browsers will load the SVG.
    Cons! SVG can't be manipulated with JavaScript.
  • iframe
    позволяет применять скрипты.
    < iframe src="triangle.svg" width="500" height="500" sandbox>
    <img src="fallback.png"/>
    </iframe>
    используется когда нет необходимости добавлять интерактивность.
    Pros! из соображений безопасности нельзя использовать картинку с другого домена
  • SVG inline
    позволяет применять скрипты.
    <svg width="300" height="200">
    <rect width="100%" height="100%" fill="green">
    </svg>
    Pros! You can use JavaScript, animation and you can style with CSS. also you can use any SVG presentation attribute as a CSS property.
    Cons! Extra SVG code increases the size of your HTML file.
    Cons! этот метод не позволяет поддерживать кэширование.
